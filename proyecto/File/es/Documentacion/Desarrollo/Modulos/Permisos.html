La gestión de los permisos se realiza según roles, el rol administrativo que es el que tiene el administrador por defecto, permite cualquier acción.

Roles por defecto:

  • administrador: Se le permiten todas las acciones.
  • usuario: Limitado

Formas de uso:

permiso('editar_contenido', TRUE);

En este caso la acción a realizar esta relacionada con la edición de contenido, en caso de que el usuario actual no tenga permiso se enruta con aviso de que no tiene permisos.

El valor TRUE del segundo parametro es el que nos indica que debe enrutarse en caso de tener permisos, si no pasamos este parametro simplemente la función nos devolvería un FALSE y seriamos nosotros los que gestionasemos la respuesta.
