enrutar

Una vez realizados todas las acciones sobre cambios en el contenido nos vamos a la pagina nueva.

Por ejemplo en caso de un cambio de nombre al recargar la página daria un error por no encontrar el contenido. Con este sistema permitimos que haya dos pasos:

1) el cambio de nombre del contenido y 2) la recarga de la página para que vaya al nuevo contenido.

Entre el paso 1 y 2 los módulos pueden realizar sus acciones pertinentes.

Con este sistema evitamos que al recargar la página se repitan acciones indeseadas.

Parámetros:
$eEvento que recibimos de Eventos
$argsArgumentos posibles
